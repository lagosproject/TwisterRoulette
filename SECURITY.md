# Security Policy

We take the security of **Twister Roulette** seriously. Thank you for helping us keep the application secure.

## Supported Versions

Only the latest release of Twister Roulette is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| >= 1.0  | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**DO NOT open a public GitHub issue for security vulnerabilities.**

If you discover a security vulnerability (such as exposed credentials, dependency vulnerabilities, or insecure handling of runtime data), please report it responsibly:

1. Send an email to **security@example.com** (replace with your official security contact address).
2. Include a detailed description of the vulnerability.
3. Provide step-by-step instructions (with a Proof of Concept, if possible) to reproduce the issue.
4. Let us know if you would like public attribution once the issue is resolved.

We will acknowledge your report within 48 hours and work with you to patch the issue before making it public. We ask that you give us a reasonable window (usually 30–60 days) to address the issue before disclosing it.
